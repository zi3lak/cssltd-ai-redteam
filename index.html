The problem

AI tools speed up pentesting. They also speak to servers you don't control.

A tester pastes a host, a finding, a credential fragment into a public LLM to get a faster write-up or a second opinion on an exploit chain. That prompt now sits on a third party's infrastructure — outside the engagement's confidentiality terms, outside your NDA, and arguably outside what UK GDPR calls data minimisation. Most firms respond by banning AI tools outright, and lose the speed. A smaller number use them carelessly, and take on risk nobody signed off on.

The stack — two systems, one perimeter

A

CSS Privacy Gateway

A de-identifying reverse proxy that sits between your team and any external LLM. One egress IP, one account — the provider sees a single anonymous origin, never your analysts, never your clients.

  • Secret redaction — API keys, private keys, high-entropy tokens stripped irreversibly before egress
  • Reversible pseudonymisation — client, product, host and domain names swapped for placeholders outbound, restored inbound
  • STRONG mode — AST-level identifier renaming that preserves code semantics, native for Python, tree-sitter for JS/TS/JSX/TSX/Go/Rust
  • Per-client rate limits and daily token budgets, with overrides
  • Streaming-safe — pseudonymisation holds even when a secret splits across SSE chunk boundaries
  • Full observability — Prometheus metrics, a live Grafana dashboard, structured audit logging
B

cssltd-redteam

A fine-tuned model that runs entirely on infrastructure you control. For the routine 80% of questions — methodology, tool syntax, report phrasing — nothing ever needs to leave the room at all.

  • Fine-tuned on curated CSSLTD red-team methodology and engagement-report style, iterating with every training round
  • Grounded in real technical corpora, not model memory — full Kali Tools catalogue and Linux manual pages, indexed and retrievable
  • Benchmarked against the base model on every release by an independent judge model, not by feel
  • Deploys on-premise or air-gapped — no engagement data ever needs an internet route
